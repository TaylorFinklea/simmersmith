SimmerSmith Privacy Policy

This policy describes how the current SimmerSmith app and service layer handle data. It is written for the product in this repository as it exists today.

Summary

SimmerSmith is a self-hosted meal planning app. It does not require a built-in account and does not include advertising, analytics SDKs, or third-party crash reporting in this repository. The iPhone app stores some data locally for offline use and syncs with the SimmerSmith server you configure.

Data stored on your device

  • Your configured server URL
  • Your bearer token, stored in the iOS Keychain
  • A local cache of profile data, current week data, recipe lists, recipe metadata, and export snapshots for offline reading
  • Local grocery check-off state for grocery items
  • In the browser admin surface, a local theme preference may be stored in browser local storage

Data sent off device

The app sends data to the SimmerSmith server you choose to connect to. Depending on how that server is configured, this can include recipes, meal plans, grocery items, household preferences, feedback, export payloads, and recipe imports.

SimmerSmith can also expose AI-related server features. In this repository, AI is routed through the server rather than directly from the iPhone app. If the server operator enables external AI providers, recipe or planning content may be sent from the server to those providers. That behavior depends on the server configuration, not the app alone.

Permissions and access

  • Camera: used only when you choose to scan a printed or handwritten recipe into a draft.
  • Photos: used only when you choose an image from your photo library for recipe import.
  • Files: used only when you choose a PDF recipe to import.
  • Network access: used to communicate with the SimmerSmith server you configure.

The current app does not request microphone, location, contacts, calendar, or health permissions.

Notifications, reminders, and Apple integrations

The current iPhone app does not request notification permission and does not schedule local or remote notifications.

This repository does include export payloads for Apple Reminders, but the actual Reminders write step is handled by a separate host-side helper outside the iPhone app. The app itself does not request Reminders permission.

Accounts, analytics, ads, and personal data

  • No built-in user account system is required by the app in this repository.
  • No advertising SDKs are included.
  • No third-party analytics SDKs are included.
  • No third-party crash reporting SDKs are included.
  • The developer does not receive your meal-planning data unless you intentionally send it in a support request.

If you email support, your email address and the information you include in that message will be received and used to respond to your request.

Contact

For privacy questions or requests, contact support@finklea.dev.
